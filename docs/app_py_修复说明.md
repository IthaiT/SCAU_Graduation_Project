# app.py 全面修复说明

## 1. 推理与评估口径修复

- 新增“推理数据范围”选项，支持 `test / val / train / all`，默认 `test`。
- 模块二不再直接使用“全量尾部窗口”，改为先按训练口径切分再做窗口推理。
- 使用 `SEQ_LEN=60` 的统一窗口约束，自动限制有效回测天数，避免窗口长度越界。
- 模型对比图与指标计算均基于当前选择的数据范围，避免评估口径混乱。

## 2. 归一化与训练一致性修复

- 将切分比例统一为训练配置：`TRAIN_RATIO=0.72`、`VAL_RATIO=0.10`。
- 新增 `create_inference_context`：
  - 先按时间顺序切分 `train/val/test`；
  - 仅用 `train` 拟合 `scaler_x` 和 `scaler_y`；
  - 推理和反归一化统一复用该上下文，消除“推理阶段重新拟合scaler”的偏差。
- `sliding_window_inference` 改为严格校验特征列，不再对缺失列静默补零。

## 3. 特征对齐修复

- 将 `FEATURE_COLS` 顺序改为与 `final_data.csv` 对齐：
  - `north_momentum` 放在时间特征之前；
  - 保证模型输入维度和语义顺序与训练一致。
- 新增 `load_aligned_feature_data`，直接读取 `data/final_data.csv` 并按 `FEATURE_COLS` 取列。
- 新增“加载训练对齐数据”按钮作为主入口，避免在线数据与训练分布错位。

## 4. 数据质量与鲁棒性修复

- `process_all_features` 改为严格模式：若关键特征缺失直接报错，不再静默填 0。
- 模型权重不存在时不再继续推理，改为跳过该模型并提示。
- 单步预测中若 `LSTM_Transformer` 权重缺失，立即中止并给出错误提示。
- 示例数据补齐 `amount` 列，避免示例路径与模型输入要求不一致。

## 5. 指标与可视化修复

- 修复 MAPE 的除零风险：对分母加入安全下限。
- 修复布林带列名不一致问题：
  - 使用 `bbl_5_2.0_2.0 / bbm_5_2.0_2.0 / bbu_5_2.0_2.0` 绘图，
  - 与实际特征工程输出一致。

## 6. 交互与文案修复

- 侧边栏数据操作调整为三类入口：
  - 加载训练对齐数据；
  - 获取最新数据并构建特征；
  - 使用示例数据。
- 首页使用说明更新为新的工作流，强调默认 test 评估与训练对齐数据优先。

## 7. 验证结果

- 语法检查通过：`python -m py_compile app.py`。
- `python test_app.py` 在当前环境失败，原因是运行环境缺少 `pandas` 依赖，并非本次代码语法错误。

## 8. 在线数据与代理兼容性修复

- `fetch_stock_data` 改为使用项目内置的多数据源拉取器 `fetch_and_clean_data`，并在请求阶段临时清理代理环境变量，降低 `ProxyError` 概率。
- 在线获取失败时不再抛出异常导致应用崩溃，改为提示用户切换到“加载训练对齐数据”。
