# AI 学习笔记

## 一、张量形状与训练流程

### 1. `(B, 30, 17)` 是什么？

一个三维张量的形状，描述**一个 batch 喂给模型的数据**：

| 维度   | 含义            | 说明                                                  |
| ------ | --------------- | ----------------------------------------------------- |
| **B**  | Batch Size      | 一次喂给模型的样本数（如 32）                         |
| **30** | Sequence Length | 每个样本回看过去 30 个交易日                          |
| **17** | Features        | 每天 17 个指标（开高低收、均线、RSI、MACD、布林带等） |

一个 batch 的数据量 = 32 × 30 × 17 个数值。

---

### 2. 滑动窗口：数据怎么变成样本？

**不是切面包（不重叠），而是滑动门（每次挪 1 天）：**

```
样本1: [第1天 ── 第30天] → 预测第31天
样本2:  [第2天 ── 第31天] → 预测第32天
样本3:   [第3天 ── 第32天] → 预测第33天
...
```

窗口之间重叠 29 天，所以 **4827 行数据 → 约 4797 个样本**（而非 4827 ÷ 30 = 160 个）。

核心代码（`src/data/dataset.py`）：
```python
total = len(data) - seq_len - pred_len + 1
self.X = [data[i : i + seq_len] for i in range(total)]  # i 每次 +1，滑动窗口
```

---

### 3. Batch vs Epoch：数据不会"用完"

| 概念           | 含义                               | 本项目数值                                  |
| -------------- | ---------------------------------- | ------------------------------------------- |
| **1 个 Batch** | 取 32 个样本喂给模型，更新一次参数 | 32 个样本                                   |
| **1 个 Epoch** | **所有**训练样本过一遍             | 3837 ÷ 32 ≈ 120 个 batch                    |
| **全部训练**   | 重复多个 Epoch                     | 100 个 Epoch × 120 batch = 12000 次参数更新 |

```
Epoch 1:  [batch1] [batch2] ... [batch120]  ← 全部数据过完第1遍
Epoch 2:  [batch1] [batch2] ... [batch120]  ← 再过第2遍
...
Epoch N:  早停机制触发，训练结束
```

**每个 Epoch 都会重新使用全部训练数据，数据不会"用完"。**

---

### 4. Batch Size 影响的是什么？

不影响用多少数据，影响的是**多久更新一次参数**：

| Batch Size | 每 Epoch 更新次数 | 特点             |
| ---------- | ----------------- | ---------------- |
| 1          | 3837 次           | 更新频繁，不稳定 |
| 32         | ~120 次           | 平衡稳定性和效率 |
| 全部       | 1 次              | 最稳定但太慢     |

---

### 5. 为什么要多个 Epoch 反复看同样的数据？

- 第 1 遍：模型几乎瞎猜，误差大
- 第 10 遍：开始捕捉规律
- 第 50 遍：预测越来越准
- 第 80+ 遍：可能过拟合（背答案），**早停机制**（`patience=20`）在此时终止训练

---

## 二、LSTM 原理

### 1. LSTM 的核心思想

用**三道门**控制信息的记住和遗忘，解决普通 RNN 的梯度消失问题（读到第 300 页忘了第 1 页）。

### 2. 三道门 + 记忆传送带

**遗忘门**（旧记忆里哪些该丢？）：
$$f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$$
- 输出 0~1：0 = 完全忘记，1 = 完全保留

**输入门 + 候选记忆**（新信息里哪些值得记？）：
$$i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)$$
$$\tilde{C}_t = \tanh(W_C \cdot [h_{t-1}, x_t] + b_C)$$
- $i_t$：写入比例（0~1）；$\tilde{C}_t$：新内容草稿（-1~1，用 tanh 因为需要有正有负）

**记忆更新**（灵魂公式）：
$$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$$
- **加法**让梯度可以长距离传播 → 解决梯度消失

**输出门**（记忆里哪些部分要对外暴露？）：
$$o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)$$
$$h_t = o_t \odot \tanh(C_t)$$
- 不是所有记忆都需要输出，有些信息现在不重要但以后可能有用

### 3. 公式中各符号含义

| 符号             | 含义                                      |
| ---------------- | ----------------------------------------- |
| $x_t$            | 当前时间步的输入（如第 t 天的 17 个指标） |
| $h_{t-1}$        | 上一步的隐藏状态（短期记忆/输出）         |
| $C_{t-1}$        | 上一步的细胞状态（长期记忆）              |
| $W$, $b$         | 可学习的权重和偏置（四套独立的）          |
| $\sigma$         | Sigmoid，压缩到 (0,1)，用作"开关比例"     |
| $\tanh$          | 压缩到 (-1,1)，用于生成有正有负的值       |
| $\odot$          | 逐元素相乘                                |
| $[h_{t-1}, x_t]$ | 拼接向量，让门"同时看到过去和现在"        |

### 4. 为什么用 2 层 LSTM？

- **第 1 层**：提取低级模式（价格涨跌、量价背离等简单特征）
- **第 2 层**：在第 1 层基础上提取高级模式（趋势反转、多指标复杂组合关系）
- **不用 3 层以上**：数据仅 ~4800 样本，层数太多会过拟合，2 层是经典平衡点

### 5. LSTM 是循环计算的

同一个 LSTM Cell 被**重复使用** 30 次（共享权重），每次把 $h$ 和 $C$ 传给下一步：

```
x₁ → [Cell] → h₁ → x₂ → [Cell] → h₂ → ... → x₃₀ → [Cell] → h₃₀
```

一个 batch（32 个样本）会**并行**产出 32 个 $h_{30}$，打包为 `(32, 64)` 张量。

## 三、关键网络组件

### 1. Dropout：随机关掉神经元防过拟合

训练时随机关掉一定比例的神经元（如 20%），逼迫每个神经元独立学会有用特征，防止"死记硬背"。预测时全部打开。

**哪里加、哪里不加：**

| 加 Dropout 的地方         | 原因                   |
| ------------------------- | ---------------------- |
| LSTM 层间                 | 参数多，容易过拟合     |
| Transformer MHA/FFN 内部  | 参数量大，过拟合重灾区 |
| 全连接层 Linear→ReLU 之后 | 参数多                 |

| **不加**的地方   | 原因                               |
| ---------------- | ---------------------------------- |
| LayerNorm        | 没有大量参数，不需要正则化         |
| 取 last step     | 只是选向量，不是计算层             |
| Concat           | 只是拼接                           |
| **最后一层输出** | 丢掉任何神经元都会直接损害预测精度 |

**原则：中间层信息冗余 → 加；输出层信息瓶颈 → 不加。**

### 2. LayerNorm：特征维度标准化

对每个样本的每个时间步，在特征维度上做标准化：

$$\text{LayerNorm}(x) = \gamma \cdot \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta$$

- $\mu$、$\sigma^2$：64 维向量的均值和方差
- 标准化后均值≈0、方差≈1
- $\gamma$、$\beta$：可学习参数，让模型自己决定最终分布

**作用**：LSTM 输出的不同维度数值范围差异大，直接送 Transformer 会导致大数值维度主导 Attention。LayerNorm 把所有维度拉到同一量级，让 Transformer "公平听到"每个维度。

### 3. 取 last step：提取序列最终总结

LSTM 输出 `(B, 30, 64)` 中，`h₃₀` 理论上包含全部 30 天信息：

```
h₁ = f(第1天)
h₂ = f(第1~2天)
...
h₃₀ = f(第1~30天)  ← 全部历史的"总结"
```

**局限**：实际上遗忘门会丢失早期信息，$h_{30}$ 对第 1~5 天的信息保留很弱。

**Serial 模型的弥补方案**：分成两条路——左路取 last step 保留 LSTM 的"最终判断"，右路把全部 30 步送 Transformer 重新审视，两路 Concat 取长补短。

| 模型            | 压缩方式                             | 思路                               |
| --------------- | ------------------------------------ | ---------------------------------- |
| ① Serial        | last step + Transformer 补救         | 两条路互补                         |
| ② Parallel 右塔 | Global Average Pooling（30步取平均） | 不偏向任何时间步                   |
| ③ 论文          | last step → reshape → mTransformer   | Attention 在特征维度重新分配重要性 |
