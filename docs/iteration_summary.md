# 模型迭代过程总结（精简版）

## 一、总体思路

本项目围绕沪深300指数预测任务，对 LSTM、Transformer 及其混合架构进行了五轮迭代。整体优化路径并不是盲目调参，而是按照“发现问题、定位原因、针对修复、再次验证”的方式逐步推进。

迭代主线可以概括为：

- V1：先严格复现论文结构，验证原始混合模型是否可用
- V2：先解决训练不稳定和模式坍塌问题
- V3：再解决串行结构下短期特征被过度平滑的问题
- V4：进一步处理金融数据中的异常值和噪声透传问题
- V5：补全评估指标与可解释性，使结果更符合论文答辩要求

---

## 二、各版本迭代过程

## V1：论文严格复现版

这一阶段的目标是尽量按照论文原始描述搭建 LSTM-Transformer 融合模型，并与纯 LSTM、纯 Transformer 基线进行对比。

实验结果表明，两个基线模型虽然能够稳定收敛，但方向准确率都在 50% 左右，说明它们更像是在复制前一天价格，落入了“随机游走陷阱”。而融合模型表现更差，出现了明显的训练失败：验证集 loss 在后期发散，模型预测输出接近一条水平线，最终 MSE 高达 138467，DA 只有 44.60%。

这一轮最重要的结论是：原始串行 LSTM-Transformer 架构在当前金融时序任务上不能直接稳定训练。问题根源不在于数据无法学习，而在于 LSTM 输出分布不稳定，进入 Transformer 后导致注意力异常、梯度爆炸和优化震荡，最终引发模式坍塌。

---

## V2：训练稳定性修复版

V2 的核心目标不是立刻提升指标，而是先让融合模型能够稳定训练，不再输出常数。

针对 V1 的问题，我做了三项关键修复：

- 在 LSTM 输出后加入 LayerNorm，稳定输入 Transformer 前的特征分布
- 在训练中加入梯度裁剪，抑制梯度爆炸
- 将融合模型学习率降为 5e-4，并改用 CosineAnnealingLR，使优化过程更平滑

这一轮优化后，融合模型的水平线现象被完全消除，预测曲线重新出现真实波动，DA 从 44.60% 提升到 50.37%。这说明模式坍塌问题已经被解决。

不过，V2 也暴露出新的问题：虽然模型不再崩溃，但三个模型的 DA 仍都在 50% 附近，说明模型仍然没有真正学到稳定的方向信息；此外，融合模型的 MSE 依旧高于两个基线，验证 loss 后期仍有振荡。

因此，V2 的意义在于打通训练链路，为后续结构优化奠定基础。

---

## V3：特征融合与跳跃连接版

在 V2 中，我发现模型虽然能训练，但普遍存在“滞后预测”现象，也就是预测结果近似于真实价格的右移一步版本。这说明模型主要是在拟合价格水平，而不是学习价格变化方向。

进一步分析后，我意识到问题出在纯串行结构上。LSTM 提取到的短期动量特征，在送入 Transformer 后很可能被自注意力的平滑效应削弱，导致最终输出缺乏短期有效信息。

因此，V3 的核心改动是引入特征融合：

- 保留 LSTM 最后一个时间步的输出
- 同时取 Transformer 最后一个时间步的输出
- 将两者拼接后送入最终全连接层
- 同时加入 `weight_decay=1e-3`，缓解验证集振荡

这一轮之后，融合模型的 DA 提升到 50.70%，首次超过基线模型，说明它开始学习到一些超越随机游走的方向信息。

但 V3 仍然没有完全解决误差问题。虽然方向预测能力有所提升，融合模型的 MSE 仍高于纯 LSTM，而且原始拼接方式本质上仍然是线性组合，无法有效过滤 Transformer 分支中的噪声。

---

## V4：鲁棒损失与门控融合版

V4 主要解决 V3 留下的两个瓶颈：

- MSE 对金融数据中的异常值过于敏感，容易产生极端梯度脉冲
- 线性拼接融合层缺乏非线性筛选能力，噪声特征容易直接传到输出层

为此，我做了两项关键改进：

- 将训练损失从 MSELoss 改为 HuberLoss，提高对异常值的鲁棒性
- 将原来的单层线性融合升级为带瓶颈结构的门控融合网络：`Linear(128,64) -> ReLU -> Dropout -> Linear(64,1)`

这一步的意义在于，模型不再只是“把两路特征拼起来再线性求和”，而是能够先压缩信息、再进行非线性筛选，从而保留更有价值的特征、抑制噪声分支。

V4 对最终性能提升非常关键，它使融合模型开始在误差控制和方向预测两个方面同时逼近甚至超越基线，是后续最终版本的性能基础。

---

## V5：学术定稿与可解释性增强版

到了 V5，模型核心结构已经基本稳定，因此这一轮不再大改架构，而是重点补全论文答辩所需要的评估完备性和可解释性。

主要工作包括：

- 在原有 MSE、RMSE、MAPE、DA 基础上补充 MAE 和 $R^2$
- 修改模型输出接口，暴露动态门控中的 `gate_lstm` 和 `gate_trans`
- 在评估脚本中加入门控权重提取与可视化逻辑

这样做的价值有两点。

第一，评估体系更完整。加入 MAE 和 $R^2$ 后，模型性能不再只看平方误差和方向准确率，而是能够从一阶误差、二阶误差、相对误差、方差解释力等多个维度综合评估。

第二，模型从黑盒变成了可解释系统。通过分析门控权重，可以看到模型在不同隐藏维度上究竟更依赖 LSTM 的短期特征，还是 Transformer 的全局特征。这一点对于论文答辩和学术表达非常重要。

根据十轮基准测试记录，最终的 Parallel-LSTM-Transformer 在综合表现上最好，说明整个迭代过程最终收敛到了一个性能、稳定性和可解释性都较为平衡的版本。

---

## 三、五轮迭代的核心结论

整个项目的迭代过程可以总结为以下几点：

- V1 证明了原始论文结构在本任务上不能直接稳定工作
- V2 解决了训练不稳定和模式坍塌问题
- V3 通过特征融合解决了短期特征被过度平滑的问题
- V4 进一步增强了模型对异常值和噪声的鲁棒性
- V5 则让整个系统具备了更完整的评估体系和白盒可解释性

换句话说，我的迭代逻辑并不是简单地“不断加模块”，而是每一轮都围绕上一轮暴露出的关键问题做针对性修复。最终得到的模型，不只是一个预测结果更好的网络，更是一个训练稳定、结构合理、指标完整、可用于论文答辩展示的研究系统。

## 四、简要版本总表

| 版本 | 主要问题                     | 核心改动                                          | 结果                         |
| ---- | ---------------------------- | ------------------------------------------------- | ---------------------------- |
| V1   | 模式坍塌，输出水平线         | 严格复现论文原始结构                              | 融合模型训练失败，DA 44.60%  |
| V2   | 方差偏移、梯度爆炸、优化震荡 | LayerNorm + 梯度裁剪 + 低学习率 + CosineAnnealing | 坍塌消除，DA 提升到 50.37%   |
| V3   | 串行结构过度平滑短期特征     | 特征拼接融合 + weight decay                       | DA 提升到 50.70%，首次超基线 |
| V4   | 异常值敏感、噪声透传         | HuberLoss + 门控瓶颈融合                          | 综合性能进一步提升           |
| V5   | 评估不完整、缺少可解释性     | 补充 MAE/$R^2$ + 暴露门控权重                     | 形成最终论文答辩版           |
