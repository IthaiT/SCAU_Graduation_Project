# V7 重构报告：二分类流水线

## 概述

将原有 **三分类（跌/平/涨）** 全面重构为 **二分类（跌/涨）** 流水线，彻底消除数据泄漏。四种模型架构（LSTM、Transformer、LSTM-Transformer、Parallel-LSTM-Transformer）现已运行在一条零前瞻偏差的数据管线上。

---

## 1. 数据泄漏消除

### 1.1 移除 CEEMDAN（关键修复）

**问题**：旧版流水线在训练/测试划分**之前**对整条时间序列执行全局 CEEMDAN（完全集合经验模态分解与自适应噪声）。CEEMDAN 利用 *整条序列*（包含未来测试数据）来确定模态边界，导致严重的 **前瞻偏差**：每个训练样本都包含了来自未来观测值的信息。

**修复**：从 `features.py` 和 `generate_features.py` 中**彻底移除** CEEMDAN。所有特征现在仅使用回溯型 Pandas-TA 技术指标计算：

| 类别   | 新增指标                                    |
| ------ | ------------------------------------------- |
| 趋势   | SMA(5)、SMA(20)、EMA(12)、EMA(26)、MACD     |
| 动量   | RSI(14)、随机指标、Williams %R(14)、ROC(10) |
| 波动率 | 布林带、ATR(14)                             |
| 成交量 | OBV（能量潮）                               |

每个时刻 $t$ 的指标**仅依赖** $t$ 及之前的数据，零未来信息。

### 1.2 替换全局 MinMaxScaler

**问题**：旧代码虽然仅在训练集上拟合 `MinMaxScaler`，但其参数（最小/最大值）是**静态**的——无法适应金融数据的体制变化。当测试集分布发生偏移（金融数据中这是必然的）时，缩放效果退化，并从训练期间隐式引入了分布假设。

**修复**：替换为**滚动 Z-Score 标准化**：

$$z_t = \frac{x_t - \mu_{t-w:t}}{\sigma_{t-w:t}}$$

其中 $w = 60$（交易日，约3个月）。核心特性：

- **严格因果**：时刻 $t$ 仅使用 $[t-w, t]$ 区间的数据。
- **自适应**：自然处理非平稳性与体制变化。
- **无训练/测试边界依赖**：每个样本自归一化。
- **预热期**：前 `min_periods=20` 个样本使用扩展窗口，之后丢弃剩余 NaN 行。

## 2. 二分类标签：数学论证

### 2.1 定义

预测目标为 **未来5日累计收益率**：

$$r_t^{(5)} = \frac{P_{t+5} - P_t}{|P_t|}$$

$$y_t = \begin{cases} 1 & \text{若 } r_t^{(5)} > 0 \quad \text{（涨）} \\ 0 & \text{若 } r_t^{(5)} \leq 0 \quad \text{（跌）} \end{cases}$$

### 2.2 为何选择二分类而非三分类

1. **"平"类是噪声**：旧版基于阈值的"平"类（$|r| < 0.5\%$）阈值设定具有任意性，且占据标签分布的约44%，造成严重的类别不平衡，同时不产生任何可操作的交易信号。

2. **市场微观结构**：0.5%的日收益率阈值未考虑波动率周期变化。在 VIX=10 和 VIX=30 的环境下，0.5%的变动含义截然不同。

3. **实践价值**：二分类方向预测可直接映射为做多/做空决策。"平"类预测在交易场景中无法执行。

### 2.3 为何选择5日预测窗口

- **降噪**：单日收益率被微观结构噪声主导。5日窗口可平滑日内噪声，同时保持足够短的战术操作周期。
- **交易成本相关性**：5日持有期将交易成本分摊至多天，使预测具备实调操作性。
- **统计特性**：周度收益率具有更优的分布特性（更接近高斯分布）。

### 2.4 标签与特征对齐

滚动 Z-Score 丢弃前约60行，5日前瞻标签丢弃后5行。有效数据范围精确对齐：

```
原始序列:      [0 ────────────────────────────── N]
Z-Score后:     [~60 ─────────────────────────── N]   (前~60行丢弃)
标签范围:      [0 ────────────────────────── N-5]     (后5行丢弃)
有效范围:      [~60 ─────────────────────── N-5]      (交集)
```

最终数据集：**4,802 条有效样本 → 3,361 训练 / 480 验证 / 961 测试**（70/10/20 时序划分）。

## 3. 架构变更

### 3.1 输出层

四个模型的 `num_classes` 默认值从 3 → **2**。输出 logits 形状：`[B, 2]`。损失函数保持 `nn.CrossEntropyLoss`（适用于任意类别数）。

### 3.2 Parallel 模型门控重设计

**问题**：旧版门控对 LSTM 和 Transformer 双塔使用独立的 Sigmoid 输出：
```python
# 旧版: gate_weights ∈ [0,1]^{2H}，拆分为两个独立掩码
gate = nn.Sequential(Linear(2H, 2H), Sigmoid())
gate_lstm, gate_trans = gate_weights.chunk(2)
fused = gate_lstm * feat_lstm + gate_trans * feat_trans
```

存在问题：
- 两组门控可能同时饱和（同时趋近0或1），导致路由失效或冗余计算。
- 深层 Sigmoid 链在输入较大时遭受梯度消失。

**修复**：互补门控 + ReLU 隐层：
```python
# 新版: 单一门控 g ∈ [0,1]^H，互补路由
gate_net = nn.Sequential(Linear(2H, H), ReLU(), Linear(H, H), Sigmoid())
g = gate_net(concat)
fused = g * feat_lstm + (1 - g) * feat_trans
```

优势：
- **互补性**：每个维度 $g + (1-g) = 1$——正确的软路由，无死维度。
- **ReLU 中间层**：避免门控计算路径中的梯度消失。
- **可解释性**：$g_i \approx 1$ 表示第 $i$ 维偏向 LSTM；$g_i \approx 0$ 表示偏向 Transformer。

### 3.3 激活函数升级

Transformer FFN 层的 `ReLU` 替换为 `GELU`，提供更平滑的梯度流（现代 Transformer 文献标准做法）。

## 6. 破坏性变更与迁移指南

1. **必须重新训练**：旧 `.pth` 权重不兼容（输出层维度从 3 → 2）。运行 `python scripts/train.py` 重新训练。
2. **必须重新生成特征**：运行 `python scripts/generate_features.py` 重建不含 CEEMDAN 列的 CSV。
3. **`get_dataloaders()` 签名变更**：现接受 `df: pd.DataFrame` 而非 `df_values: NDArray` + `columns: list[str]`。
